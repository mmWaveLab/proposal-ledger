### 文案

本项目拟购置 Radxa Cubie A5E 迷你主板，选定规格为商业级 A527 SoC / 4GB 内存版本，用作雷达、心音、心电多模态数据采集系统的边缘主控与网络转发节点。官方资料显示，Cubie A5E 可选全志 A527 SoC 或工业级 T527 SoC，搭载 8 核 Arm Cortex-A55 CPU；A527 版本支持 LPDDR4/LPDDR4x 内存、microSD/eMMC/M.2 NVMe 存储、2 个千兆以太网口、Wi-Fi 6、蓝牙 5.4、USB 3.0、USB Type-C OTG、40 Pin GPIO，以及 M.2 M Key 2230 扩展。

拟建系统面向雷达生命体征信号与心音、心电同步采集实验，用于支持算法验证、数据集构建、设备联调和后续原型系统迭代。双千兆网口便于将雷达设备网络与实验室上位机/局域网分离，降低网络冲突和带宽争用；USB 3.0 与 USB Type-C OTG 可用于连接声卡、串口转换器或采集外设；40 Pin GPIO 可用于触发、同步、I2C/SPI/UART 控制；M.2 NVMe 和 microSD/eMMC 存储可用于采集过程中的边缘缓存和离线备份。

## 采购理由

- 支撑雷达/心音/心电多模态同步采集系统搭建。
- 作为边缘主控节点，减少采集电脑直接连接多设备造成的端口、驱动、权限和网络配置复杂度。
- 双千兆网口适合雷达数据网络与上位机/实验室局域网分离，提高联调稳定性，并为后续网络隔离实验预留条件。
- USB 3.0、USB Type-C OTG、40 Pin GPIO 和 M.2 NVMe 扩展覆盖数据接入、触发同步、控制通信和本地缓存四类需求。
- 8 核 Cortex-A55 处理器适合运行采集守护进程、轻量滤波/质量检查、时间戳记录和数据转发服务。
- 体积小、功耗相对可控，适合桌面实验、移动采集箱和原型系统集成。

## 使用计划

1. 完成主板系统烧录、网络配置与远程管理环境配置。
2. 接入雷达设备网络，验证数据流接收、缓存和转发。
3. 接入心音/心电采集链路，验证采样时间戳记录和文件归档。
4. 建立基础采集脚本，形成“实验编号-受试者编号-模态-时间戳”的数据命名规则。
5. 形成一次完整联调记录，补充到本 README 的复盘记录中。
