### 文案

本项目拟购置 Radxa Cubie A5E 迷你主板，用作雷达、心音、心电多模态数据采集系统的边缘主控与网络转发节点。该主板具备 8 核 Arm Cortex-A55 处理器、双千兆以太网、USB 3.0、M.2 2230 扩展、Wi-Fi 6/蓝牙 5.4 等能力，适合承担便携式采集场景下的数据汇聚、时间戳记录、轻量预处理、边缘缓存与局域网传输任务。

拟建系统面向雷达生命体征信号与心音、心电同步采集实验，用于支持算法验证、数据集构建、设备联调和后续原型系统迭代。双网口设计便于将雷达设备网络、实验室采集电脑或局域网隔离连接；USB 与 GPIO 扩展便于连接声卡、ADC、同步触发或控制模块；本地存储扩展可用于采集过程中的临时缓存和离线备份。

## 采购理由

- 支撑雷达/心音/心电多模态同步采集系统搭建。
- 作为边缘主控节点，减少采集电脑直接连接多设备造成的端口、驱动和网络配置复杂度。
- 双千兆网口适合雷达数据网络与上位机/实验室局域网分离，提高联调稳定性。
- 具备 USB、GPIO、M.2 等扩展能力，便于后续接入声卡、心电采集模块、同步触发模块和本地缓存存储。
- 体积小、功耗相对可控，适合桌面实验、移动采集箱和原型系统集成。

## 使用计划

1. 完成主板系统烧录、网络配置与远程管理环境配置。
2. 接入雷达设备网络，验证数据流接收、缓存和转发。
3. 接入心音/心电采集链路，验证采样时间戳记录和文件归档。
4. 建立基础采集脚本，形成“实验编号-受试者编号-模态-时间戳”的数据命名规则。
5. 形成一次完整联调记录，补充到本 README 的复盘记录中。

## 验收标准

- 主板能够稳定启动并完成网络连接。
- 至少完成一次雷达数据与心音/心电数据的同步采集流程验证。
- 采集数据能够按实验编号自动归档。
